# Frontend Coding Rules & Guidelines (TOEIC Dictation)

You are an expert Frontend Developer working on the **TOEIC Dictation** project. When generating, refactoring, or reviewing code in the `toeic-elearning-fe` repository, you **MUST** strictly adhere to the following rules:

## 1. Architecture: Feature-Sliced Design

- **`src/app/` is strictly for Routing & Layouts:** Keep `page.tsx` and `layout.tsx` files as thin as possible. These files should primarily act as wrappers that import a "Feature Component" (e.g., `<LessonListFeature />`). **Never** place heavy business logic, `useEffect` hooks, or complex `useState` here.
- **The Core is in `src/features/`:** Every business domain (`auth`, `lessons`, `dictation`) gets its own folder inside `src/features/`. Each feature folder internally encapsulates its own `components`, `hooks`, `store`, and `types`.
- **`src/components/` is for UI Only:** Only base, generic, and reusable "dumb" components (like those from Shadcn UI) go here. Do not place feature-specific components here.

## 2. API & Data Fetching (TanStack Query + Orval)

- **Do not write manual `axios` or `fetch` requests:** All endpoints and types are automatically generated from the Backend's Swagger setup via Orval.
- **Use Generated Functions:** Import functions explicitly from `src/api/generated/` (e.g., `authControllerLogin`) and wrap them in **TanStack Custom Hooks** (`useQuery`, `useMutation`) inside `src/features/[feature]/hooks/`.
- **Data Types:** Always import Request DTOs and Response Interfaces from `src/api/generated/model/`. Avoid re-declaring API types manually.

## 3. State Management

- **Local Form/UI State:** Use `useState` or React Hook Form.
- **Complex Domain State:** Use **Zustand** for complex, multi-component states (e.g., the Dictation Typing Engine, Audio Player State, User Session). Place Zustand stores inside `src/features/[feature]/store/`.

## 4. Styling & Tailwind CSS

- Use **Tailwind CSS** for all styles. Do not use CSS Modules or inline styles.
- **Class Merging:** Always use the `cn` utility function from `@/lib/utils` when combining Tailwind classes conditionally.
- **Dark Mode Support:** Keep dark mode in mind when writing Tailwind classes (e.g., `bg-white dark:bg-zinc-900`).

## 5. TypeScript Practices

- Enable Strict Mode behaviors.
- **No `any`:** Under no circumstances should you use `any`. Use `unknown` if the type is truly unknowable, but prioritize importing exact types generated by Orval.
- Separate types into `types/` folders within features if they are frontend-specific.

## 6. Code Style & Cleanliness

- Keep components small and specialized. Move helper logic into custom hooks.
- Prefix custom hooks with `use` (e.g., `useDictationEngine`).
- Prefix event handlers with `handle` (e.g., `handleStatusChange`) and their corresponding props with `on` (e.g., `onStatusChange`).
